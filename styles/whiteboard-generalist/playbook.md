# 风格包 · Whiteboard Generalist（公用白板 · 兜底）

> 从 KuleshovAgent 迁入（2026-07-24）。机械硬门在本目录 `contract.json`（护栏面持有，本文不复述阈值）；本文管创作方法，审美要求交 Judge。
> **公测兜底包**——路由低置信、混合/未知、或诉求超出现有风格包（pixel-chronicle 像素科普 / case-file 案卷调查）预期的选题，一律落本包，保证下限。

## 0. 档案

- 状态：**兜底包（generalist fallback）**——职责是"接住其它包接不住的"，不走考片制上架（考片挣上限，兜底保下限）。
- 适用：其他 / 混合 / 低置信度 / 未知公测输入；用户诉求超出现有风格包预期时的默认收口。
- 唯一硬不兼容：写实真人重演（真人只允许符号化角色 / 叙事替身——三包共同规则）。
- 交付承诺类型：由本片合同与所选生成风格决定（见 §1）。
- 考片记录：无。

## 1. 链路配方（硬 · 三链路架构 2026-07-24）

**空白链路** = **生成风格二选一（拼贴 collage / 像素 pixel，同片单一）+ 实拍视频 + HTML + 图片 + 数字人**，元素可自由组合，唯一铁律是**生成镜风格一片一种、不得混**。

- 开工 / 分镜阶段先在 `meta.style_notes.generated_style` 写死 `pixel` 或 `collage`——选定后逐镜按该风词表执法（选 pixel 就禁 collage 词、反之亦然），保证同片单一风。
  > ⚠️ `STYLE_LOCK` 机器门 molly 暂未覆盖（见 `contract.json` 的 `_molly_enforcement`）——当前由本纪律 + Judge 执法，不是自动拦截。
- 选 pixel 走图先行像素锚点组装；选 collage 走空色场首尾帧拼贴组装——工艺随所选风。**选 pixel 即禁用 collage b-roll 声部**（其工艺就是拼贴）。两种风格下 seedance 镜均**图先行**（≥1 静帧锚 + ≥10s 慢组装，见 `contract.json` 的 `traits.craft`）。
- 实拍视频（`footage`）、静态图片（`gpt_image`）、数字人（`avatar`，须挂 portrait 锚点 + 音频先行）都可用；HTML 做简单排版，版式经 compose 门。

## 2. 定位（先读这段，防包名误导）

- **本包不是"白板画风"**——"白板"指的是**给你一块空白画布的质量下限模板**：一套与画面外观无关的生产纪律（每镜单一传播意图 / 参考资产 / 连续性 / 边界设计 / 统一手柄 / 实测验收），套在任何长尾选题上保证片子不塌底。
- **画面外观完全由选题决定**：做菜的片该有食物质感，讲财报的片该是数据版式——AI 视频 / 真实素材 / 图表 / 图片动效 / 数字人 / 文字包装自由混排。把任何选题都做成白底手绘涂鸦风，是对本包最常见的误读，视同模板味（Judge 专项直接打）。
- **唯一硬不兼容**：写实真人重演（真人只允许符号化角色 / 叙事替身，三包共同规则）。

## 3. 必须成立的东西（Judge 验收，合同只查声明在场）

全片必须有统一的色彩 / 字体 / 节奏**或**重复视觉手柄——开工时把手柄声明写进 `meta.style_notes.visual_handle`（合同机械门查在场性，是否成立交 Judge）。**换个选题仍完全成立的模板化画面 = 不合格**（Judge 模板味专项）。

## 4. 生产纪律（通用视频生产纪律的原则化，不机械照搬）

1. **锁定 brief**：开拍后不改目标；改目标 = 回 brief 重走。
2. **重复人物与关键物建参考资产**，固定引用映射（`@Image1 = …` 一经分配不换序）——见 produce SOP 锚点阶段（anchors）。
3. **显式维护跨片段连续性**：服装 / 朝向 / 道具位 / 光线逻辑逐镜检查。
4. **片段边界设计**：可读的结束构图 + 下一镜切入手柄。
5. **生成后 ffprobe 验证真实时长**（`tools/measure-render.py` 成片实测层做，自报与实测矛盾以实测为准）——按实测时长排片。

## 5. 素材配比（硬门 · storyboard 门前移拦截）

**纯 HTML/MG 动画（`provider=hyperframes`）按时长份额 ≤ 40%**；剩余 **≥60% 必须由真素材声部承担**，从下列自由混排：

- **AI 图片动效**（`image_motion`）：GPT-Image 生成图 + 运镜 / 视差，做示意图、场景、情绪空镜；
- **真实图片 / 真实视频**（`footage`）：Pexels / 档案实拍空镜、真实照片；
- **数字人**（`avatar`）：口播人设、旁白具象化；
- **seedance 视频**（`seedance` / `collage_broll`）：决定成片上限的高价值镜头、拼贴 b-roll。

这条在 **storyboard 阶段**由 `style.contract.plan` 门按声明的 `shots[].source.provider` 时长份额判定（`plan.provider_share.hyperframes.max`）——**花钱和出片前**就挡住"全 MG 幻灯片"。分镜时就要把镜头价值想清楚、主动把 6 成以上时长分给真素材，而不是默认全塞 hyperframes 等 render 期被打回。hyperframes 该承担的是：动态字幕 / 字卡、数据卡、UI 演示、过场衔接——不是整片主视觉。

## 6. 动效升级（"做得更炫酷 / 更有动效"落这里，不靠堆特效）

用户提"更炫酷 / 更有动效 / 像发布片"这类模糊诉求时，**不要随机叠效果**——走 compose 知识包的 `.claude/skills/produce/references/hyperframes-recipes.md`（F01–F16 few-shot）：

1. **先定叙事动作**：出现 / 替换 / 完成 / 比较 / 交互 / 进入另一个世界 / 展示并行规模；
2. **选 1 个主配方，最多叠 1 个辅助**（配方决策顺序见该文件 §2）；
3. **品牌变量只换皮肤，不改 motion grammar**：字体 / 颜色 / 圆角 / 纹理 / 资产按选题替换，时序结构照抄；
4. **炫酷 ≠ 模板**：套用后要能写清用了哪些配方、对应什么叙事动作、关键时间点——换个选题仍完全成立的模板化画面 = 不合格（Judge 模板味专项）。

引擎工艺（怎么写才不崩：属性 / 媒体 / 子 composition / 确定性）在 `.claude/skills/produce/references/hyperframes.md`，本包不复述，只负责"按选题气质选对表达、保住下限"。

## 7. 传播

- **搜索词先行**：选题 / 标题含 1 个用户真的会搜的词，长尾问句化优先。
- **钩子节律**：3 秒钩住 → 15 秒内亮核心承诺 → 每 15–30s 一个小高潮。
- **首帧海报化**：hook 镜首帧按海报标准设计（可直接充当封面资产）。
- 医疗 / 心理 / 法律等强监管选题：只讲风险 / 法规 / 机制，不给场景化专业建议；绝对化用语禁绝（广告法通杀项）。
- 引用平台机制数字必须标 [官方规则] / [社区共识] / [推测]。

## 8. 兜底路由

EP 在 produce 开拍提问（`SKILL.md` §1.3）按选题推荐风格包 → 置信不足或包间冲突 → 落本包；风格 × 选题不兼容（`incompatible_topics`）时在剩余合法包里改道，全不合法则停点问人。
