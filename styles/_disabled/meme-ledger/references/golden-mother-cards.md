# Golden Few-shots：十篇母稿的质量标杆

> 每张卡对应一份用户提供的**完整 ASR 原文 few-shot**，路径与完整性校验记录在 `golden-source-library.json`。使用时先读本卡选母稿，再完整读取已选原文，最后读 `human-speech-engine.md` 的相应路由说明。生成时必须重建事件材料与全部措辞，不能把原文当作句子词库。

## G01｜营养工厂，暴打保健品

- **来源**：BV1QjhWzmEcC｜约 07:59｜约 3,025 字
- **质量标杆**：先把行业的荒诞与不透明讲透，再让“透明后来者”以可比较的事实登场；不是单向带货。
- **程序**：真实需求 → 包装/价格反差 → 恐惧与供应链机制 → 逻辑击穿 → 新解法 → 最强反方 → 人的行动 → 行业回收。
- **事实槽位**：消费者恐惧、成熟供应链、参数/价格、报告/售后、真实起源或行动。
- **质量检查**：新玩家必须有可核实的数字或做法；没有就不许使用“后来者破局”段。

## G02｜活着就是熬着，活下去就是一切

- **来源**：BV1yB4y1Q7ZN｜约 11:54｜约 4,571 字
- **质量标杆**：不保证成功，仍然给出一个不失真的最低行动；情绪靠具体成本而不是励志口号。
- **程序**：三连现实问题 → 个人失败 → 运气与长期窗口 → 可观察的重复动作 → 情绪拉高再设安全边界 → 同代账单 → 尺度扩大 → 回到最低动作。
- **事实槽位**：失败细节、时间跨度、日常动作、群体压力、真实结尾人物。
- **质量检查**：不能冒充第一人称经历；不能把“熬”写成对制度问题的推卸。

## G03｜瑞幸咖啡是如何暴打资本主义的？

- **来源**：BV1T7411W7ca｜约 14:40｜约 5,456 字
- **质量标杆**：同一笔钱在不同座位里含义相反；先让观众理解最动人的商业故事，再逐项结算代价。
- **程序**：反常消息与最低知识 → 多本账 → 增长公式 → 最好听的故事及反例 → 谁相信 → 产品真实价值 → 财务数字 → 新故事续命 → 各方结算。
- **事实槽位**：消费者收益、管理层/资本动机、公开财务、同构反例、新业务或第二幕。
- **质量检查**：没有公开财务与多方账本，不能硬套成“资本故事”。

## G04｜5本让你怀疑世界的好书

- **来源**：BV1eZ4y1j7kw｜约 10:01｜约 3,853 字
- **质量标杆**：清单的每一项都把同一命题推深；压轴项让观众从听推荐转为亲自判断。
- **程序**：免责声明 → 第一视角（解释/应用/反证） → 相邻两项构成台阶 → 日常尺度切换 → 压轴困境与连续追问 → 定义权回收。
- **事实槽位**：每项的核心视角、现实迁移、局限、压轴困境的多种合法答案。
- **质量检查**：五项不能只并列；没有递进或压轴综合，就不是这个母版。

## G05｜你知道奶茶加盟到底有多坑人吗？

- **来源**：BV1UJ41157Lw｜约 16:53｜约 6,769 字
- **质量标杆**：从“听起来能赚钱”还原为一条逐步锁死退出的付款生命周期，同时给观众出口。
- **程序**：主体核验 → 名字不等于公司 → 正规不等于适合 → 五问自测 → 沿付款链的八坑 → 终局激励变化 → 替代路径 → 高门槛重定义。
- **事实槽位**：主体/合同、收费节点、每个节点的品牌收益与用户沉没成本、可执行替代方案。
- **质量检查**：不能凭个案断言骗局；法律或监管入口必须按实际项目核验。

## G06｜买不起奢侈品就是穷人吗？不，是聪明人

- **来源**：BV1x7411i7au｜约 17:18｜约 7,113 字
- **质量标杆**：先尊重欲望，再拆解身份定价，最后给规则；批评不是羞辱消费者。
- **程序**：营销话术与边界 → 立刻可做的防线 → 选择权重定义 → 身份如何给商品定价 → 精准溢价 → 供应链证据 → 让步 → 消费算法 → 安全感 → 新的“酷”。
- **事实槽位**：功能/审美/身份的区分、可核验供应链或安全差异、量化规则、替代方案。
- **质量检查**：不得假设高价产品同质；让步段不可删除。

## G07｜为何漠视人命

- **来源**：BV1cC4y1x7TA｜约 12:32｜约 4,696 字
- **质量标杆**：把看似混乱的反常决策压缩成一条激励因果链，并追踪到普通人承担的成本。
- **程序**：反向夸奖与事实 → 决策模型 → 核心激励 → 短期指标 → 三根长期资产 → 逐根受损 → 现实压力 → 资源分配 → 人的底线。
- **事实槽位**：决策时间线、公开激励/表态、短期指标、长期资产、具体成本承担者。
- **质量检查**：只能从文件、合同、公开表态推断动机；不能把推断写成内心事实。

## G08｜热点事件谁是赢家

- **来源**：BV1gE41147rZ｜约 09:57｜约 3,499 字
- **质量标杆**：每找出一个“赢家”，就把赢家门槛抬高一次；最后回到最早人物完成循环。
- **程序**：双方皆失 → 候选赢家一 → 更高标准后的候选二 → 平台受益 → 最像正解的公司候选 → 明确标注的巧合推演 → 回到开场人物。
- **事实槽位**：各方具体损失、流量/财务/代言或商业联系、可区分事实与喜剧推演的材料。
- **质量检查**：角色必须多于两方；时间相邻不能伪装成真实因果。

## G09｜为什么操控者屡屡得手

- **来源**：BV1jE41137PK｜约 15:09｜约 5,679 字
- **质量标杆**：先定义并排除，再用“需求—包装—留存—变现”解释有效性，不把被影响者写成愚蠢。
- **程序**：边界定义 → 识别需求 → 产品包装 → 可谈判的底线 → 满足心理缺口 → 低/高客单变现 → 多操控者相遇的市场结果。
- **事实槽位**：真实需求、策略差异、警觉成本、退出与防御方法。
- **质量检查**：不得受害者归责；高风险题材必须提供安全退出信息。

## G10｜东方众神暴打华尔街韭菜

- **来源**：BV1t7411b7F7｜约 20:46｜约 7,264 字
- **质量标杆**：先用准确的浅解释建立规则，再用多个独立案例反复让规则撞墙；每个失败都留下讽刺性的后传。
- **程序**：统一世界观 → 机制一/案例一/后传 → 机制二/案例二 → 新机制打断 → 后传升级 → 最长主线案例 → 重复触发器三连 → 闭环。
- **事实槽位**：至少三个独立案例、各自机制、事实反差、角色动机与明确标注的合成/虚构后传。
- **质量检查**：只有一个案例时禁止使用；每个案例先讲规则再讲荒诞。

## Golden 质量评分

每期在 TTS 前对照所选 Golden 打分，满分 25，低于 21 或任一项低于 3 分不得投产：

| 维度 | 5 分标准 |
| --- | --- |
| 结构忠实度 | 段落任务、相对篇幅、转折和结尾均由所选母版推出 |
| 事件等价性 | 每个母稿槽位都有真实的新事件材料，不是硬塞名词 |
| 人话翻译 | 抽象机制都有具体场景、选择、比例和普通人落点 |
| 事实与反方 | 所有数字可回链；最强反方被公平表达后才反转 |
| 节奏与回收 | 母比喻递进回收，问题/列举/留白的位置服务推理 |
