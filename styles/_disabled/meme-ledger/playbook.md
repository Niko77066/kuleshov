# 风格包 · Meme Ledger（梗账本）

> 候选包。定位：面向抖音/小红书的商业、互联网与消费机制解读。口播负责把机制讲清，画面只用项目已登记合作授权的 UnusualVideos 频道视频做语义反差；HyperFrames 不加任何效果，只负责确定性装配。

## 0. 档案

- 状态：**候选**（已完成一次试跑，尚未通过独立风格验收）。
- 唯一视频源：[UnusualVideos YouTube 频道](https://www.youtube.com/@UnusualVideos)。仅在项目已经登记合作授权时可使用；每个选段都须在资产账本登记原视频 URL、video ID、入点、出点、用途、授权引用与本地文件，以便追溯、复跑和替换。
- 适用片型：`faceless_explainer`；有可核验事实、可拆利益/激励链、且观点允许带黑色幽默的商业与互联网议题。
- 不适用：事故灾难、个体受害者、医疗/法律建议、仍在事实核查中的指控；这些内容里 meme 会稀释真实伤害。
- 交付承诺类型：`licensed_channel_footage + static_caption + voiceover + bgm`。画幅由 brief 指定；所有可见底画面都是频道原视频，字幕是唯一允许的视觉叠加层。时长由文案母稿决定：只有用户明确要短稿，或母稿本身是短稿时，才压缩到 60–100 秒。
- 考片记录：一次横版试跑证明装配链可用，但文案结构验收未达标。下一条须经 `/video-score`、结构复盘与版权清单双重验收后才可转正。

## 1. 风格宣言

1. **口播把账算清，画面负责让人停下来。** 事实与推理完整地活在 research 和口播中，频道视频不替它们背书。
2. **meme 是平行隐喻，不是论据。** 它负责给抽象机制一个动作和意外，不得被字幕包装成事实现场。
3. **笑点来自尺度错配。** 高概念落到具体生活物件，离谱清单第三项越界，但不靠热词堆砌。
4. **快，不乱。** 一句只服务一个判断；只用硬切，切点发生在句读、重音、画面动作或 beat 上，留白用于让 punchline 落地。
5. **零包装。** 不加转场、滤镜、颗粒、调色、缩放、平移、变速和动效字；源视频本身就是视觉语言。

## 2. 开拍提问模板（elicitation）

1. 这件事的反常识表象是什么？请用一句话说出“谁在什么约束下，以什么机制，得到了什么反常结果”。
2. 利益账本里有哪些座位：消费者、平台/公司、员工、供应商、监管者？钱、注意力或风险如何流动？
3. 哪 3 个事实已经可核验？每一条对应的原始来源是什么？频道画面不得被当作这些事实的证据。
4. 观众能从哪个日常物件理解这个机制？它可以成为全文反复回收的母比喻吗？
5. 这个题材是否涉及真实受害者或正在发展的指控？若是，删去 meme 反应位，改用克制的事实片型。

## 3. 文案引擎：逐段结构移植，而非“像某种风格”

写作前必须读取 [references/golden-mother-index.json](references/golden-mother-index.json)、[references/golden-mother-cards.md](references/golden-mother-cards.md)、[references/golden-source-library.json](references/golden-source-library.json)、[references/human-speech-engine.md](references/human-speech-engine.md) 与 [references/f-of-x-transplant-design.md](references/f-of-x-transplant-design.md)。前三者分别负责母稿检索、few-shot 质量标杆与用户提供的完整 ASR 原文；选定母稿后必须完整读取对应原文，再进入事件映射。`human-speech-engine.md` 负责“场景、选择、比例、普通人”的翻译动作；`f-of-x-transplant-design.md` 规定不得把原文压成模板摘要后再由通用语言补齐。选定 G08 时还必须读取 [references/g08-surface-prose-profile.md](references/g08-surface-prose-profile.md)，把表层行文当成硬约束，而不是事后“润色”。

### 3.1 对味的原因

手机涨价 V6 的有效之处不在于更密的梗，原因是它恢复了成熟母稿真正起作用的四层结构：

1. **段落程序没有被压扁。** 每一段都有唯一任务，且任务顺序不跳跃：日常入口 → 欲望/反差 → 机制奇观 → 商业反方 → 历史/常识类比 → 争议留白 → 普通人情绪落点 → 收束。
2. **篇幅跟着修辞任务走。** 机制、反方和生活落点必须拥有足够篇幅；不能把它们压成“事实一段、比喻一段、结论一段”的报告。
3. **母比喻不断升级。** 同一物件第一次解释机制，第二次解释利益排序，第三次解释争议，最后才收回普通人的处境。它不是一句口号。
4. **事实槽位逼着每次转场发生。** 数字、角色、反方和情绪都从事件事实推出；meme 只给节拍，不代替推理。

### 3.2 强制写作路线

当用户提供了对标原稿、授权转写或明确的结构对照文件时，必须采用**逐段结构移植**。禁止先写一个通用“商业吐槽模板”，再往里面塞事实。

1. **锁定一篇主母稿。** 必须从已建立的十篇母稿中按叙事发动机选择一篇，并记录至少两个被排除的候选及理由；不拼接多篇，确认其适配的叙事几何。沿付款/交付/沉没成本生命周期拆账才可选《奶茶加盟》；没有连续收费链的服务机制题必须转去《营养工厂》或其他匹配发动机，不能因为“有多个角色”就硬套收割链。
2. **拆出段落程序。** 为每段标记时间占比、修辞动作、情绪位置、问题/列举位置、母比喻承担的任务与信息槽位。
3. **先补齐事件事实。** 每一个母稿槽位都必须有新事件的已核实事实、明确推断或可用的生活观察；没有等价材料就删改该槽位并在验收中记录，绝不硬造。
4. **逐段建映射。** 创建四列对照：`母稿段落作用 / 原段摘要 / 新稿对应段 / 保留的结构项`。先确认映射，再写口播。
5. **按原顺序扩写。** 保持段落功能、相对篇幅、问题密度、转折位置和情绪坡度；替换事件、人物、数字、比喻载荷与全部措辞。
6. **最后才配视频。** 按已冻结口播寻找动作对应、预期落空或尺度错配的频道选段；不得为了迁就现成梗改变文案推进。
7. **双重验收。** 检查结构同构与事实可追溯；结构相似不等于可以复用原句、独特口癖或真人声线。

### 3.2a Anti-Smooth Gate：先验拒绝，而不是写完再润色

段落数、字数、比喻密度相近，不能证明新稿来自同一叙事发动机。写作时必须保留原稿的“下一段为什么只能从这里出来”的接力关系；具体协议见 [references/f-of-x-transplant-design.md](references/f-of-x-transplant-design.md)。

1. 选定母稿前，先画新事件的**主体—动作—可见物/数字—后果—下一节点**材料图。
2. 每个相邻段必须有接力证明：`上一段末尾的具体裂缝 → 下一段必须出现的具体对象/动作`。回答“逻辑上也该讲”一律视为断链。
3. 第 8 篇《肖战事件谁是赢家》必须同时拥有四个可点名候选、每人真实动作、连续失败理由、至少一条历史/时间关系和可标记为荒诞的回环材料；缺一项即拒绝该母稿，换题或换母稿。
4. 首稿禁止提前写最终结论；候选必须以“专名 + 动作”入场，禁止用“第一类赢家 / 平台 / 影院 / 消费者”这样的类别词凑人物。
5. 六项 Anti-Smooth Gate 任一项不通过，禁止进入润色、Golden 打分与 TTS。

### 3.3 十篇母稿路由（先选发动机，后写文案）

| 母版 | 选择条件 | 不该用在什么情况 |
| --- | --- | --- |
| 营养工厂 | 成熟供应链、信息不透明，有透明/低价的新解法 | 没有可核实的后来者或可比较成本 |
| 活着就是熬着 | 普通人的长期困境，不能保证逆转但能保留行动 | 需要即时解决的安全/专业问题 |
| 瑞幸咖啡 | 同一商业结果在消费者、公司、资本等账本中意义相反 | 缺乏公开财务或多方博弈 |
| 5 本好书 | 多个对象能构成同一认知台阶 | 单一事件或凑数清单 |
| 奶茶加盟 | 沿付款/交付生命周期拆收割点 | 没有明确收费链 |
| 奢侈品 | 身份欲望、品牌溢价、消费选择 | 没有身份定价或功能差异真实显著 |
| 川普与人命 | 反常组织决策可还原为“短期激励 → 长期资产 → 普通人成本” | 无证据就揣测个人动机；复杂事件只有情绪没有因果链 |
| 肖战事件 | 至少四个具体角色的动作能彼此牵出，连续抬高“谁是赢家”的标准，并有可标记的回环材料 | 只有两方；只有抽象利益相关者；商业受益不可核实；下一候选不能由上一段事实逼出 |
| 渣女得手 | 本质是识别、包装、留存、变现的操控漏斗 | 受害者归责或安全风险未处理 |
| 东方众神 | 多个独立案例、机制复杂、角色众多的竞争/泡沫 | 只有一个案例，或只能靠虚构人物撑段落 |

**强制规则：**“群像多”不等于该用《东方众神》；必须同时具备多个独立、可核验案例。单一公司决策即使角色很多，也优先判断是否属于“激励因果链”。“有多个环节”也不等于《奶茶加盟》：只有钱款或沉没成本在生命周期中连续转移时才可采用它。

### 3.4 对标原稿的专用长稿骨架（不是默认模板）

下列八段只适用于用户提供的 `benchmark-vs-v5-structural-transplant.md` 对标原稿，或段落程序经验证与其同构的母稿。它不是所有长稿的默认形状。原稿决定实际时长；除非用户明确要求压缩，不能自行把它缩成一分钟。

| 段 | 修辞动作 | 新事件必须提供的材料 |
| --- | --- | --- |
| 01 | 日常刚需 → 二分法 → 母比喻 → 痛点 | 一个高频生活入口、反常价格/规则、可贯穿全文的具体物件 |
| 02 | 个人经验 → 价值欲望 → 工具反差 | 普通人为什么在意、只“能用”为什么不够、一个具象荒诞入口 |
| 03 | 密集列举 → 体验对照 → 行业判断 | 至少三条核实事实/数字，以及它们共同指向的机制 |
| 04 | 最强商业反方 → 核心答案 → 成年困境 | 反方最合理的解释、成本如何传导到普通人 |
| 05 | 看似不划算 → 历史/常识类比 → 不投入的诱因 | 一个不靠煽情的类比，以及为什么各方会拖延/转嫁 |
| 06 | 争议留白 → 价值结论 | 事实边界、反例或预测的不确定性；不能写成单一反派 |
| 07 | 电影化日常 → 现实打断 → 母比喻回收 | 普通人场景、不能被解决的现实、短促而独立的结论 |
| 08 | CTA 槽位 | 频道自有 CTA；不挪用对标创作者的固定口令 |

### 3.5 短稿路线

只有在用户明确要求 60–100 秒时，才能压缩为“钩子 → 一条关键事实 → 一个机制比喻 → 反方 → 收束”。压缩时仍要先完成完整母稿映射，再选择连续的核心段落；不得拿七段短模板冒充长稿移植。

建议口播：270–320 汉字/分钟，平直、清楚、略带不敢置信的停顿；不模仿任何具体真人声音或口癖。

默认 TTS：MiniMax `speech-2.8-hd`，使用项目级已授权声音档，参数见 `voice-profile.json`。共享风格包不得写入真人身份、复刻音色 ID 或参考音频；项目在 `rights/` 中证明声音使用/复刻授权后才可生成。全文 single-pass 生成，改一个字即重跑整轨与强制对齐；笑点由句读和画面承担，音色不加夸张笑声或角色表演。

## 4. 视觉系统：源视频 + 静态字幕

- 底画面：只使用指定频道的视频选段；保留原始色彩、帧率和画面运动，不做 LUT、滤镜、颗粒、锐化、模糊、暗角或压缩噪点。
- 适配：默认保持源画面比例，以 `contain/letterbox` 适配 9:16，避免为填满竖屏而裁掉笑点主体；确需裁切时只能做一次静态构图裁切，并在镜头账本记录 crop 参数。
- 字幕：参考帧的朴素白色粗体 + 黑色描边/黑底安全区；字幕位置固定，不做逐字弹跳、关键词变色、扫光或入场动画。每屏 1–2 行，以手机端可读为准。**每句末尾不加任何标点**（`。！？… .!?`）；停顿由时间轴与切点表达。
- 黑边：允许作为源比例适配的结构空间，不额外放标题、角标、贴纸、进度条或结论卡。
- HyperFrames：只允许素材时间窗、尺寸适配、硬切、静态字幕、音频混合和响度归一；禁止任何视觉效果或动画。

## 5. 节奏、剪辑与声部表

- 翻译总表引用：`干脆 / snappy`；本包覆写为平均 20–28 cuts/min，单个频道选段通常 0.6–2.5 秒。笑点需要动作完成时可放宽到 4 秒，但不能用冻结帧补时长。
- 切点：同时满足三者中的至少两个——口播句读/重音、源视频动作落点、BGM 强拍；不能为了“快”在一句中间切断因果。
- 转场词汇只有一种：硬切。白闪、故障擦除、溶解、推拉、遮罩、变速衔接全部禁用。
- 每 10–14 秒允许一个动作较完整、节奏较松的频道选段作为呼吸；不插黑底结论卡或证据卡。

| 声部 | 固定叙事角色 | 允许来源 | 份额/规则 |
|---|---|---|---|
| `licensed_channel_footage` | 全片底画面：动作隐喻、预期落空、升级、反转、情绪呼吸 | 仅 UnusualVideos 频道合作授权视频 | 画面时长占比 100%；至少 6 个选段；不承载事实主张 |
| `static_caption` | 逐句口播字幕 | HyperFrames 静态文本层 | 唯一允许的视觉叠加；固定位置、无动画 |

## 6. BGM 规则：项目曲池，带种子的随机选择

每个项目先在 `rights/` 冻结 2–4 首已获授权的候选曲，再从其中等概率随机选择一首；随机结果与整数 seed 写入 `audio.music.selection_seed`，确保同一项目复跑不换歌。共享风格包不保存商业曲名、链接或未验证的授权主张。

- 版本：只使用纯器乐、无对白/说唱/人声的高记忆点片段；选定 8–24 秒可无缝循环的段落，记录源文件、片段入出点与 loop crossfade 参数。
- 选择：`seeded_uniform_random`，不按题材主观挑歌，不因“这首更搭”静默改结果；要换歌必须更换 seed 并记 ledger 决策。
- 混音：旁白始终第一；BGM 目标比旁白低约 18–22 LU，成片整体 -14 LUFS、true peak ≤ -1.5 dBTP。
- 权利：用户本次确认的是频道视频合作授权，并未一并声明四首商业录音的同步权/母带权。生产项目必须在 `audio.music.license_ref` 登记实际所用版本的权利证明；缺证明可以完成预览，但不得进入 `deliver`。

## 7. 反模式（一票否决）

- 使用指定频道以外的任何底画面，或频道选段缺原 URL、video ID、入出点与授权引用；
- 把频道视频当证据，或让字幕把无关画面伪装成事实现场；
- 使用真实受害者、灾难、疾病或普通人的窘境当 punchline；
- “一个句子一个梗片”的机械乱切；选段必须有动作或语义关系；
- 任何结论卡、证据卡、贴纸、角标、动效字幕，以及 LUT、滤镜、颗粒、缩放、平移、变速、白闪、故障、溶解等效果；
- 使用未登记对应同步权与母带授权证明的商业 BGM 交付，或让音乐盖住旁白；
- 直接模仿在世创作者的固定口癖、个人声音或独特句子。

## 9. 写作出厂门

文案进入 TTS 前必须同时满足：

1. `research.md` 逐条标明事实、推断、观点与不可写项；
2. `script.md` 存在段落对照表，且每个主段有明确的结构任务；
3. 结构移植项目写明主母稿、两个被排除母稿、授权/来源状态、主动偏离项与时长理由；
4. 母比喻回收至少 4 次，每次解释新的关系；
5. 任何事实连续说明超过 3 句后，必须进入追问、具体物件、类比或转折；
6. 不出现对标稿的独特句子、固定口令、真人声线或可识别口癖。
7. 通过 `references/human-speech-engine.md` §6 的六问复核；任意两问答不上来，退回结构映射阶段。
8. 按 `references/golden-mother-index.json` 的质量门提交 Golden 评分，低于 21/25 或任一维低于 3 分不得进入 TTS。
9. 通过 `references/f-of-x-transplant-design.md` §5 的 Anti-Smooth Gate；对于 G08，六项必须全绿，并在项目中保存逐段接力证明。
10. 若主母稿为 G08，通过 `references/g08-surface-prose-profile.md` §4 的表层硬门；任何可脱离专名复用的句子，或无事实解锁的 AI 胶水句式，都不得进入 TTS。
11. 用户指定短版时，标记为 **compressed adaptation**：记录被保留的一条连续升级链、删去的段落任务与删减理由。短版不得声称为完整母稿移植，也不得用“钩子—事实—机制—结论”的通用说明文补齐缺口。
12. 商业机制题至少写出三次可回链的利益/风险转移（谁付钱、谁获利、谁承担风险或退出）；每次幽默比喻必须引出一个事实推断、转场裂缝或情绪落点，不能只负责逗笑。
