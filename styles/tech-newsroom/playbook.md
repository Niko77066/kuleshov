# 风格包 · Tech Newsroom（科技新闻深访）

## 0. 档案

- 状态：**草案**（诞生于《Samsung Health AI 同意条款》一片的开拍提问，用户口述方向："新闻视频、网页实录 + 数据流 MG + 医疗/隐私 AI 场景 + 科技新闻数字人"。未考片，本包是待验证假设——每条都等作者裁决）
- 适用片型：`faceless_explainer`（科技/隐私/政策解读）、深访型 news_recap
- 交付承诺类型：`mixed_source / evidence_led`——混源是本包的语法本体，来源切换即声部切换

## 1. 风格宣言

1. 像一段**严肃科技媒体的深访报道**（The Verge / 晚点 一类的解释性报道），不是营销片也不是快报。
2. **证据优先**：说到条款就上条款原文（网页实录），说到数据就上数据卡（MG），说到影响就上场景（生成视频），下结论时人出面（数字人）。声部即论证结构。
3. 屏幕内容是主角：真实页面、真实条款文字、真实按钮。**禁止伪造 UI**——网页实录必须来自真实公开页面的截取，带地址栏或出处角标。
4. 冷静、克制、追问式语气。观点靠事实排列产生，不靠形容词。
5. 恨：惊悚阴谋论腔（红色警报、骷髅图标）、科技感堆砌（矩阵雨、全息 HUD）、无出处的"据报道"。

## 2. 开拍提问模板

1. 追问的核心问题是什么？（一片只追一个问题，答不完砍范围）
2. 证据链里哪些是**可实录的真实页面**？（条款页 / 设置页 / 弹窗——没有实录证据的主张降级处理）
3. 数字人承担什么角色？（开场提问者 / 结尾裁决者 / 全程主持——默认"首尾出面，中段隐身"）
4. 场景化镜头（Seedance）表达什么抽象概念？（数据流动 / 使用场景——每片 ≤ 3 镜，防氛围镜头稀释证据密度）

## 3. 视觉系统

```css
--bg:      #101216;   /* 深空灰（主背景，非纯黑） */
--panel:   #1A1D23;   /* 面板灰（卡片/实录容器） */
--text:    #E8EAED;   /* 主文字 */
--muted:   #9AA0A8;   /* 注文/出处 */
--accent:  #4A9EFF;   /* 数据蓝（数据流、强调、下划线） */
--warn:    #FFB454;   /* 琥珀（风险点/关键条款高亮，全片 ≤ 3 次） */
--rule:    #2A2E36;   /* 分隔线 */
```

- 字体：标题与正文均无衬线（Noto Sans SC；标题 700，正文 400），数字 `tabular-nums`；条款引文可用等宽字体块以示"原文引用"；
- 网页实录容器：统一浏览器窗框（圆角 + 地址栏），实录内容不满屏、坐在面板上，出处角标常驻右下；
- 数据流 MG：节点-连线语言（圆点 + 1.5px 线 + 沿线流动粒子），一屏 ≤ 5 节点；
- 层次三级：主张 > 证据 > 出处；同屏琥珀高亮 ≤ 1 处。

## 4. 节奏与剪辑

- **旁白声音档（逐字复用）**：`一位专业的中文科技新闻主播（成年女性，普通话，冷静克制，语速中等）`（seed-audio-1.0）；
- ⚠️ **实测校准（2026-07-14，samsung-health-ai-consent take-1）**：该声音档实际读速 **~5.1 字/s（≈306 字/60s）**，且句间近零间隙——通用预算表的 220–260 字/60s 对它**不成立**。用本包写 60s 片按 **280–310 字**预算，或在 prompt 里显式要"句间留呼吸"并预留节界留白；
- TTS 语气"冷静追问"，句末不上扬，禁感叹腔；
- 声部节奏：数字人段 ≤ 8s/次；实录段 3–6s 配旁白精读；MG 段 4–8s；场景段 3–5s；
- 翻译总表引用：`克制/measured`（主行）；
- 转场词汇（≤ 4）：硬切（默认）、面板推移（声部切换）、白帧 2 帧（结论前）、缩放进实录（进入证据时）；
- J-cut 默认：旁白先行 0.2–0.4s 进下一声部。

## 5. 声部表

| 来源 | 角色 |
|---|---|
| 数字人（HeyGen） | 提问者与裁决者：开场抛问题、结尾给判断；中段隐身 |
| 网页实录（自采截录） | 证据声部：条款原文、设置路径、弹窗文案——本包的公信力来源 |
| MG 动画（HyperFrames） | 信息主轨：数据流向图、条款拆解卡、对比表、字幕层 |
| 生成视频（Seedance） | 抽象概念具象化：健康数据的采集/流动场景、使用情境（每片 ≤ 3 镜） |
| 图片动效 | 备用：新闻配图、示意插画（≤ 2 镜） |

## 5b. MG 质感三件套（2026-07-15 实测：扁平→高级的关键，参照 heygen hyperframes-launch 官方示例）

> 教训：v1 用扁平深灰底 + 硬边卡片，工整但"素"，被判"HyperFrames 能力没吃满"。以下三招把 MG 拉到官方示例质感，且不违背新闻克制（不上彩虹/矩阵雨/全息 HUD）。

1. **动态氛围背景**（死底变活底）：全片一个 `bg-ambient` 子 composition 垫底（独立 track，DOM 首位保证在最底）——深空底 `#0A0C10` + 2–3 个大半径 `blur(120px)` 径向辉光球（冷蓝 `--accent` / 深青 / 极弱琥珀 `--warn`），用 GSAP `sine.inOut yoyo` 在 20–26s 周期内极缓漂移；叠极淡网格（rgba 0.028 + 径向 mask）+ 晕影。确定性：只用有限 tween + yoyo，禁 Date.now/random。
2. **液态玻璃面板**（卡片/浏览器窗/芯片统一）：`background: linear-gradient(160deg, rgba(32,38,48,.55), rgba(16,19,25,.4))` + `backdrop-filter: blur(16–20px) saturate(1.12)` + `border:1px rgba(255,255,255,.11)` + `box-shadow: 0 30px 90px rgba(0,0,0,.5), inset 0 1px 0 rgba(255,255,255,.15)`。烘焙型场景 `.bg` 必须改 `transparent` 让氛围底透出，玻璃才有"东西可透"。
3. **对焦落定入场 + 强调色辉光**：hero 元素入场 `gsap.set({filter:"blur(18–22px)", y:30–50})` → `to({filter:"blur(0px)", y:0, ...power3.out})`（官方招牌运动）；强调用辉光而非变色——蓝线 `box-shadow:0 0 12px rgba(74,158,255,.7)`，大字 `text-shadow:0 0 32px rgba(74,158,255,.3)`，琥珀 `text-shadow:0 0 22px rgba(255,180,84,.55)`。大字表态镜可加局部 `radial-gradient` 光池。
4. **lint 陷阱**：`backdrop-filter` 玻璃在 docker 渲染正常；track-index 别和音频轨（0）撞——氛围底给独立 track；GSAP 动 scaleX/filter 的元素别在 CSS 里同时写 `transform`（用 `gsap.set` 设初值）。

## 6. 版式模板清单

1. **追问开场卡**（2–3s）：核心问题大字 + 数字人画中画或全屏；
2. **实录精读框**：浏览器窗框 + 关键句琥珀高亮 + 旁白同步下划线；
3. **数据流图**：你的手机 → 服务器 → 训练管线的节点连线动画；
4. **条款拆解卡**：左侧原文（等宽引用块）、右侧人话翻译；
5. **裁决收尾卡**（3–5s）：数字人 + 一句判断 + "条款链接见简介"角标。

## 7. 反模式（一票否决）

- 伪造或摆拍 UI 冒充实录；实录无出处角标；
- 阴谋论视觉（红光、黑客帽、代码雨）；
- 数字人全程霸屏（变口播 PPT）；
- 琥珀高亮超过 3 处（狼来了效应）；
- 无来源切换逻辑的混剪（声部必须对应论证角色）。
