# 说人话引擎：十篇母稿的共同写法

> 目标不是把十篇稿件平均成一种腔调，而是根据事件的冲突结构，选对叙事几何；再把抽象概念变成一个普通人能看见、能比较、会在意的具体场景。

## 1. 十篇母稿的底层能力

| 母版 | 它真正解决的问题 | 把抽象事翻译成人话的方式 |
| --- | --- | --- |
| 营养工厂 | 贵与好为什么常被混为一谈 | 把供应链、成本和透明度变成消费者能核对的账 |
| 活着就是熬着 | 不确定的长期困境怎么说才不鸡汤 | 不承诺胜利，只把行动压到一个还能做的最小动作 |
| 瑞幸咖啡 | 同一结果为何有人叫好、有人叫苦 | 换座位：消费者、公司、资本、对手分别算账 |
| 5 本好书 | 多个对象怎样不沦为清单 | 每一项把同一问题再推进一层，压轴项回收定义权 |
| 奶茶加盟 | 风险怎样从“听起来不错”变成可拒绝的链条 | 沿付款、交付、沉没成本的顺序，把坑排成路线图 |
| 奢侈品 | 人为什么愿意为“不是功能”的东西付钱 | 先承认欲望，再分开功能、审美、身份与选择权 |
| 川普与人命 | 反常决策为何持续发生 | 一条因果链：短期激励 → 长期资产 → 普通人成本 |
| 肖战事件 | 混乱热点里到底谁受益 | 一层层抬高“赢家”标准，直到表面答案不成立 |
| 渣女得手 | 操控为什么有效 | 用获客、包装、留存、变现的漏斗还原心理机制 |
| 东方众神 | 复杂竞争如何讲得既准又好玩 | 每轮先教一个机制，再用独立案例让规则撞上现实 |

## 2. 不是“风格”，是四个翻译动作

### 2.1 从名词翻译到场景

不要说“渠道重构”“供给错配”“品牌溢价受到侵蚀”，先问观众会在什么时候碰到它。

- 渠道重构 → “谁还能在门口拿喇叭喊折扣”；
- 供给错配 → “停车场先把车位租给愿意包月的大巴”；
- 品牌溢价受损 → “原价像一块每周都写‘最后三天’的立牌”。

场景不是装饰。它必须能解释至少三个事实，并在后文逐次升级。

### 2.2 从结论翻译到选择

“行业会受影响”不是一句能播的话。要写成谁在做什么两难选择：

- 厂商是在涨价、减配置还是牺牲利润；
- 经销商是在清库存、保流水还是守价格；
- 消费者是在马上买、等折扣还是继续忍旧设备。

选择一出现，利益、代价和立场才会自然出现。

### 2.3 从数字翻译到比例

数字必须先准确，再给日常比例。

- “线上收入占 22%”之后，解释为“最能带客流的一层不再由你经营”；
- “DRAM 合约价涨 13%–18%”之后，解释为“报价单像两根同时抬起的栏杆”。

类比只帮助理解，不替代事实，也不能把估计写成确定因果。

### 2.4 从宏大叙事翻译回人

结尾不强行宣布时代结论。回到一个已被前文证明会受影响的普通动作：下单、换机、开店、上班、删照片、看购物车。

“人话”的最后一步不是煽情，是让观众知道这件事已经落在自己哪一张账单上。

## 3. 十篇母稿共有的段落纪律

1. **先让观众认出自己，再解释系统。** 日常痛点不是闲聊，它是理解门票。
2. **一段只做一个动作。** 吓、问、拆、比、列、辩、翻、落、收；一段塞两个动作，节奏就会报告化。
3. **每次转场必须被上一段逼出来。** 不能因为作者“还有一条资料”就插入新段。
4. **连续三句事实后必须换气。** 用追问、具体物件、反方或类比，不让信息变成念数据。
5. **反方要说到最强。** 如果观众用一句“企业也有成本”就能推翻稿子，说明稿子还没开始。
6. **笑点服务理解。** 笑点只能来自事实反差、尺度错配、具体生活物件或自嘲；不能拿受害者、无证据指控或热词填节奏。
7. **结尾要回收开场的冲突。** 不是换一个更大的口号。

## 4. 母版选择的判断程序

先把事件写成一句“谁在什么约束下，以什么机制，造成了什么反常结果”，再按顺序问：

1. 这是**多方账本**、**单一决策的激励链**、**生命周期陷阱**、**认知台阶**，还是**多个独立案例的群像竞争**？
2. 事件有几个独立、可核实的事实模块？只有一个核心案例，不能选《东方众神》。
3. 事件有没有公开成本、融资、补贴或盈利资料？没有，不能硬选《瑞幸咖啡》。
4. 结尾需要的是“普通人的选择”“最低行动”“赢家闭环”还是“规律总结”？这决定母稿能否真正落地。
5. 为何至少两个相邻母版不适用？写出来，能防止“看着像”就套。

## 5. 从用户反馈得到的反模式

| 失败 | 原因 | 以后怎么拦截 |
| --- | --- | --- |
| 手机涨价 V1 | 只有机制摘要，没有母稿的段落推进和篇幅坡度 | TTS 前强制逐段映射，不通过不写正文 |
| 手机涨价 V6 前的短稿设定 | 自行把长母稿压成短视频，削掉机制、反方和人的落点 | 未获用户明确压缩要求时，时长跟母稿走 |
| 耐克—滔搏 V1 | 把成功的对标原稿误当所有题材的默认模板 | 每期必须选主母稿并记录两个排除项 |
| 套用错误的复杂群像稿 | “角色多”被误判为“案例多” | 《东方众神》必须有多个独立案例，否则转选激励因果链 |

## 6. 写完后的“人话”复核

逐段问六个问题：

1. 这一段里，观众能看见什么具体人、物或动作？
2. 这段的事实和观点，谁能回链、谁只是推断？
3. 这段为什么必须排在这里？
4. 去掉梗后，它还有解释力吗？
5. 如果观众站到最强反方的位置，下一段是否已经回答？
6. 结尾回到的普通人，是否从开场起就已经在这条因果链里？

六问中有两问答不上来，就退回到“事实槽位 / 段落动作”阶段，不进入 TTS。
