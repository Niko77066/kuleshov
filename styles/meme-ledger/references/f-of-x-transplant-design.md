# F(x) 不是摘要器：防止“结构移植”产出 AI 味的设计

## 0. 这次失败的定义

错误函数是：

`F_bad(x, facts) = 通用商业观点 + 母稿的主题标签 + 平滑口语化`

它把原文 `x` 压成“赢家逐层排除、放生活比喻、最后落普通人”的摘要。摘要丢掉了原文真正推动下一段的东西：谁做了什么具体动作、这个动作暴露了什么条件、那个条件为什么又让下一个人进场。模型只能用自己最常见的“解释—举例—总结”补齐空白，于是就有 AI 味。

正确函数是：

`F_good(x, G, B) = SurfaceProfile(x, ConstrainedTransplant(Trace(x), Graph(G), Boundaries(B)))`

- `Trace(x)`：原文逐段的叙事轨迹，不是主题摘要；
- `Graph(G)`：新事件中可核实的**人物 / 行动 / 旧关系 / 数字 / 时间点 / 物件**关系图；
- `Boundaries(B)`：事实、推断、喜剧推演分别到哪里为止；
- `SurfaceProfile(...)`：独立措辞的**受控模仿层**。它继承原文的句法起伏、叙述者站位、材料密度、转弯时机与“先过头再补证”的口播感；它无权改变轨迹，也无权用漂亮话填补材料空洞。它不是可选的最后润色。

对于 G08，必须同时读取 [`g08-surface-prose-profile.md`](g08-surface-prose-profile.md)。

## 1. F 必须保留的不是“结构”，而是下一段的必然性

对原稿每一段 `p_i`，先提取五项：

| 字段 | 问题 | 第 8 篇里的抽象例子 |
|---|---|---|
| `claim_i` | 这一段暂时让观众相信什么？ | 某人也许是赢家 |
| `material_i` | 它靠哪一个具体事实、旧梗或动作成立？ | 公开表态中的限定词、旧关系、财报、日期 |
| `crack_i` | 段尾留下哪一个不能忽略的裂缝？ | “有意向”不等于真捐；流量仍受股价约束 |
| `handoff_i` | 裂缝如何**点名**下一段的对象？ | 限定词 → 捐赠机制 → 另一个人物的旧关系 |
| `status_i` | 这是事实、可核验推断，还是明确标识的荒诞推演？ | 三者不得混写 |

只有当新事件存在 `material'_i → crack'_i → handoff'_i` 的闭环，才允许写对应段。不是“这一段也可以讲影院”，而是“上一段里哪一个事实迫使影院必须出现”。

## 2. 先验拒稿：材料图不能承载母稿，就换题或换母稿

第 8 篇不是“找四个利益相关者”。它需要：

1. 至少四个可点名的候选赢家；
2. 每人有一个**已发生的具体动作**，不是抽象身份（例如“平台”“消费者”“城市”）；
3. 每个候选的失败理由，必须在前一候选的材料里生成；
4. 至少一条旧关系、历史事件或时间节点能把链条再次拧紧；
5. 最后的荒诞推演必须从已给事实出发，并被明确标成荒诞，而不是伪装为分析。

缺任一项，`F_good` 返回 `REJECT_TEMPLATE`，而不是让语言模型拿“电影、影院、文旅、观众”这样的类别词补位。

**本次暑期档题材的判定：不适配 G08。** 它有电影、影院、文旅的分析材料，但没有四个能互相引出的具体人物/组织动作，也没有足够的关系图做回环。V2 仍然是一次失败样本：段落数相同不等于叙事发动机相同，禁止进入 TTS。

## 3. 生成器的四道工序

### 3.1 先做“材料图”，不准起文

每条事实写成：`主体 — 已发生动作 — 可见物 / 数字 / 原话 — 直接后果 — 可牵出的下一节点`。

例如不能写“影院受益于暑期档”；只能写“某院线在某日因为某片改变了什么排片 / 公告 / 价格，因此牵出谁”。没有这种节点，就没有这一段。

### 3.2 逐段建立“接力证明”

每相邻两段写一行：

`p_i 的末句裂缝 [具体名词] → p_{i+1} 必须出现的原因 [新节点]`

如果能把 `p_{i+1}` 换成任何一个行业对象而不影响理解，该段判为通用模板句，必须重找材料。

### 3.3 忠实移植，禁止提前写结论

首稿不许出现最终观点的同义句。结论必须只在母稿原本揭晓答案的位置出现。禁止用“对观众来说”“真正说明了”“归根结底”“这就是为什么”提前替观众收束。

候选角色一律用**专名 + 动作**入场；禁用“第一类赢家是……”“影院会不会是赢家”这种报告标题式入场。一个抽象机制只能在角色已经行动后被总结出来。

### 3.4 独立措辞，不做“去毛刺润色”

口播允许不匀称、绕一小圈、先说错再改、突然想起一条材料；这些必须来自事件关系，而不是随机加梗。删掉所有“优雅的解释连接词”后，若段与段仍然由接力证明相连，才可润色。

### 3.5 表层行文不是装饰，是第二个硬约束

模型最常见的失败是用“不是 A 而是 B”“你以为……其实……”“从表面看”“归根结底”“对观众来说”把推理焊平。它们不是绝对禁词，但每次出现都必须有刚刚出现的具体材料为它付费。G08 的逐段 `named-action / stance / glue / generic-sentence / callback` 标记要求见 [`g08-surface-prose-profile.md`](g08-surface-prose-profile.md)。

## 4. 不再使用的伪指标

- 段落数、字数、问号数相近：只是最低限度的外形检查，不能证明对味。
- 有生活类比：类比不是原文的发动机；它不能替代人物关系和具体动作。
- 有“赢家”反转：没有交接链的反转，只是作者换了一个话题。
- Golden 总分超过阈值：若没有逐段接力证明，分数无效。

## 5. 新的出厂门：Anti-Smooth Gate

写完后，创作者必须逐段回答，任一项为否即退回材料图：

1. 这段中的专名和动作是什么？若只有行业类别词，失败。
2. 上一段末尾哪一个具体材料逼出了它？若回答“逻辑上也该讲”，失败。
3. 它在段尾留下的裂缝，具体把谁/什么送进下一段？若没有，失败。
4. 把本段与相邻段互换，是否明显断裂？若不明显，失败。
5. 最终结论是否在最后两段之前已经被说破？若是，失败。
6. 是否为了凑角色、反转或笑点新增了不可核实的人、动机、收益或因果？若是，失败。

对于第 8 篇，六项必须全绿；不能以“母稿已知”“字数够了”“文案顺”豁免。
