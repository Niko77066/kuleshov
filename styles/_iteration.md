# 风格包进化规程（darwin 棘轮版）

> 借鉴 [darwin-skill](https://github.com/alchaincyf/darwin-skill) 的自主优化循环，嫁接到仓库现有三件套：`/video-score`（≈ rubric）、`/golden-set-regression`（≈ 考卷回归）、`/screening-review`（≈ 独立评委盲评）。
> 适用对象：`styles/` 下所有风格包 + `translation-table.md` 的行。

## 核心立场

风格包和 SKILL.md 一样，是"冻结模型的外部可训练状态"——它可以被系统地变好，但**改它的人不能自己给自己打分**（LLM 自评自改实测准确率 ~46%，乐观偏差是系统性的）。所有迭代走下面的棘轮，keep/revert 的终审权在作者本人。

## 每包必备：考题镜头（test-shots）

darwin 铁律：跳过 test prompts 直接评分 = 凭空打分。对应到风格包：

- 每包目录固定 `test-shots.json`：3–5 个考题镜头（brief + 预期效果描述），如
  `{"id":"ts01","brief":"电芯内部隔膜熔穿瞬间，微观尺度","expect":"剖面标注卡语法、热色梯语义正确、无彩虹热力图、无AI塑料光泽"}`
- 考题从该包已出厂片的**高分镜头和翻车镜头**里选（golden set 的镜头级子集）；
- 任何风格包改动后，考题镜头必须实际重新生成对比——"读文本觉得写得更好了"不算数。

## 迭代循环（单变量棘轮）

```
0. 基线：考题镜头按当前包生成，走 /video-score 相关维度打分，记入工作簿   ⏸ 作者确认基线
1. 找最弱维度 → 生成 1 个改进方案（改什么/为什么/预期提升）——每轮只改一个维度：
   只动翻译表某行 / 只动某镜头类型的提示词模式 / 只动视觉系统某项
2. 编辑 + git commit（一轮一提交）
3. 重新生成考题镜头 → 独立评分（subagent 盲评或 /screening-review 随机对照，≥2 评委共识）
4. 新分 > 旧分 → keep；否则 git revert（不许 reset --hard，失败尝试也是数据）
5. 连续 2 轮 Δ < 0.5（7 维总分）→ 触顶停手                            ⏸ 作者看 diff + 对比帧裁决
```

## 规则

1. **dry_run 要标注**：不想烧生成费时允许"读提示词推演"式干跑评估，但必须在工作簿标 `dry_run`；dry_run 比例 > 30% 的轮次宣布评估不可信。省钱可以，虚报分数不行（Rule Zero 同源）。
2. **相关簇一起看**：改提示词模式常带动翻译表条目失效——单变量指"一次一个改动意图"，受牵连的行要在同一 commit 里一起改并说明。
3. **触顶即停**："+0.15 是停手信号不是继续信号"——为微调质感反复烧生成费是 over-engineering 的视频版。
4. **两轮涨不动 → 探索性重写**：git stash 保底，换参照系从头重写 style-spec（如换叙事骨架），考题镜头对比后择优。爬坡解决不了"在平庸风格附近打转"。
5. **反例黑名单是必备件**：只写"应该做 X"没有"不要做 Y"的风格包不合格——黑名单要具体到本包禁用的运镜/转场/色彩词，G2 评委按黑名单逐条核对。
6. **从不回滚 ≠ 水平高**：darwin 早期 40 次迭代 0 revert 的教训是评分太松。如果风格包迭代从不回滚，先怀疑评分器。
7. **留痕格式**（记入视频调优工作簿，一轮一行）：
   `日期 | commit | 包名 | 旧分 | 新分 | keep/revert | 改动维度 | full_test/dry_run | 备注`
8. **每轮出一张对比卡**：旧分→新分、改动摘要、before/after 帧——月底汇报素材 + 未来机器评委的校准语料索引。

## 与既有机制的接线

- 翻译表行状态机（`[假设]→[已验证]`）不变，本规程给"转正/改数"提供程序：考题镜头验证 = 转正依据；
- `/benchmark-breakdown` 的产出（新行/新模式）以 `[假设]` 入表后，进下一轮循环验证；
- 风格包准入考片制（3–5 条样片盲评赢基线）不变，本规程管**准入后的持续进化**。
