## 0. 档案

- 状态:**候选**(未考片——本包为《800V 热失控》一片新建,从蓝图原则 + 事故调查纪录片/工程爆炸图美学推理而来,是"论证严密的假设";首片出厂后补拉片验证)
- 拉片对象:待定(候选:Real Engineering、Branch Education、B站"差评君"工程解剖段落——记 DEBT)
- 适用片型:`faceless_explainer` 的物理过程/工程机理题;硬核科普
- 考片记录:无
- 交付承诺类型:`motion_led`(真运动主导,图解混排)

## 1. 风格宣言

1. 像一部**冷静的事故调查纪录片**,不是灾难片预告——张力来自"你看懂了链条",不来自火焰堆砌。
2. **剖开来看**是本包的招牌动作:任何"里面发生了什么"都用剖面/爆炸图交代,禁止黑箱比喻糊弄。
3. 热是全片唯一主角色:一条从琥珀到白炽的温度色梯,贯穿所有来源的画面;冷却蓝只留给"正常态"与"防护"。
4. 数字必须有单位、有出处、有对比锚(800°C 要说"炼钢炉口的温度",不裸奔)。
5. 恨:恐吓式剪辑、无标注的示意图、彩虹热力图、"科技感"蓝色粒子。

## 2. 开拍提问模板(elicitation)

1. 解剖对象是什么?有没有公开的结构图/标准文档可作事实底座?
2. 过程链条有几个关键节点?(≤ 5 个,超了砍——每节点 15–20s)
3. 有没有"时间锚"?(法规时限、实测数据等能给观众体感的硬数字)
4. 危险过程题:结尾落在"工程如何应对"还是"观众该做什么"?(决定最后一节声部)

## 3. 视觉系统

```css
--carbon:  #0C0F14;   /* 碳黑(背景) */
--bone:    #E8E6E1;   /* 骨白(正文/线框) */
--heat-0:  #FFB02E;   /* 热·琥珀(升温) */
--heat-1:  #FF5A1F;   /* 热·橙红(失控) */
--heat-2:  #FFF3E0;   /* 热·白炽(峰值,仅数字/闪帧用) */
--coolant: #3D9DF6;   /* 冷却蓝(正常态/防护段专用) */
--rule:    #2A2F38;   /* 分隔线/网格 */
```

- 字体:标题 Noto Sans SC 900(工程黑体,禁衬线),正文 Noto Sans SC 400,数字与标注 `tabular-nums` + JetBrains Mono(英文/单位);
- 版式:左侧 2/3 画面主体 + 右侧 1/3 标注栏的"图纸布局"为默认;标注线用 1px 骨白引出,末端小方块;
- 留白 ≥ 20%;同屏热色 ≤ 1 处(热色出现即语义:这里在升温);
- style frame 锚点:开拍后产 1 帧"剖面 + 标注"风格帧存 `anchors/`,全片 LUT 以它为基准。

## 4. 节奏与剪辑

- 语速:中文常速 220–260 字/60s(90s 片 ≈ 330–390 字);TTS 语气"冷静纪录片解说"——克制、低沉、不表演恐慌;
- 翻译总表引用:`沉稳/权威`(主行)+ `紧迫/breaking`(仅"蔓延失控"高潮段覆写,且图片动效在该段禁用照常);
- 剪辑密度:主体段 4–6s/镜,高潮段 2–3s/镜;切点落句读,J-cut 默认;
- 转场词汇(全 4 种):硬切(默认)、白炽闪帧 2–4 帧(仅热失控触发/爆燃瞬间,全片 ≤ 2 次)、剖面擦除(MG 图解 ↔ 实景切换,章节边界)、黑场 0.3s(收尾前一次)。

## 6. 版式 / 镜头模板清单

1. **剖面标注卡**:对象剖面居左,右侧标注栏逐条点亮,标注线引出;
2. **温度大数字卡**:温度值顶满 50% 画面高,count-up,右下角小字口径与出处;
3. **链路进度条**:传播链(如 电芯→模组→整包→乘员舱)横向节点条,当前节点热色点亮,常驻或章节复现;
4. **时间锚卡**:关键时限(如"5 分钟")大字 + 法规/出处小字;
5. **对比锚字幕条**:给抽象数字配体感类比的下沉字幕条(一屏一条)。

## 7. 反模式(一票否决)

- 残骸/伤亡画面、哭喊音效等恐吓元素;
- 无标注线的纯示意图(观众不知道自己在看哪);
- 彩虹热力图(热只有一条色梯);
- 蓝色粒子流、全息 HUD 等"科技感"贴皮;
- 火焰镜头连续 ≥ 3 镜(火不是主角,链条才是);
- 数字裸奔(无单位/无出处/无对比锚)。
