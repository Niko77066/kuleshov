# 风格包 · Daily Brief（冷静快报）

## 0. 档案

- 状态：**候选**（未考片——拉片对象待定，是 M0 任务之一；本包当前是"论证严密的假设"）
- 适用片型：`faceless_news_recap` 为主，`faceless_explainer` 的数据密集段可借
- 交付承诺类型：`typography_led / data_explainer`（零成本通路原生适配）

## 1. 风格宣言

1. 像一份**排得极好的晨报**，不是电视栏目包装——信息的尊严来自版式，不来自特效。
2. 全片一套网格、一个强调色；变化靠内容推进，不靠版式换花样。
3. 数字是主角：大、等宽、干脆地滚上来，绝不弹跳。
4. 快而不喘：语速顶格，但每条之间留一次呼吸（≥ 0.4s 停顿）。
5. 恨：彩虹配色、emoji、每条一个新动画、任何"庆祝感"。

## 2. 开拍提问模板

1. 数据源是什么？（榜单 / RSS / 手动清单）抓取时间？——时效戳要进片尾角标
2. 栏目名与期号？（跨期版式必须一致，第一期即定终身）
3. 本期条目数？（建议 5 条；60s 装不下 8 条，砍）
4. 强调色随栏目固定——本栏目用什么色？（默认见下）

## 3. 视觉系统

```css
--paper:  #F4F1EA;   /* 纸白（背景） */
--ink:    #17181C;   /* 墨黑（正文） */
--accent: #C8451B;   /* 印刷红（强调,栏目可覆写） */
--rule:   #C9C4B8;   /* 细分隔线 */
```

- 字体：标题衬线（Noto Serif SC 900 / 中文报头感），正文无衬线（Noto Sans SC），数字 `tabular-nums` 等宽；
- 网格：12 栏瑞士网格，左对齐为主，留白 ≥ 24% 画面面积；
- 层次只用三级：刊头 > 条目标题 > 注文；同屏 ≤ 1 个强调色块。

## 4. 节奏与剪辑

- 语速：中文快报体 280–320 字/60s；TTS 语气"冷静通讯社"，禁感叹腔；
- 条目节奏：每条 5–8s（标题卡 2s + 数据/详情 3–6s），条目间硬切 + 0.4s 呼吸；
- 翻译总表引用：`干脆/snappy`（主行）+ `紧迫/breaking`（仅 breaking 单元覆写）；
- 转场词汇（全 3 种）：硬切（默认）、色块横扫（章节边界）、白帧 2 帧（收尾前）；
- GSAP：入场 `power4.out` 0.25s；列表 stagger 0.06s；数字 count-up 0.5s。

## 5. 声部表

| 来源 | 角色 |
|---|---|
| HyperFrames | 信息主轨（刊头/条目/数据/收尾全部） |
| 图片动效 | 每期 ≤ 2 镜的新闻配图（Ken Burns ≤ 3s），只作氛围佐证 |
| 生成视频 | 本包不用（用了就不是这个风格） |

## 6. 版式模板清单

1. **开场刊头卡**（2.5s）：栏目名大衬线 + 日期期号 + 色块横扫入场；
2. **条目卡**：序号大字 + 标题两行以内 + 来源注文小字；
3. **大数字卡**：单个关键数据顶满 60% 画面高，count-up，注文说明口径；
4. **引言卡**：一句直接引语 + 出处，左侧色条；
5. **收尾卡**（2s）：栏目名 + "明天见" + 数据抓取时效戳。

## 7. 反模式（一票否决）

- 条目换版式 / 换动画；emoji / 图标堆砌；彩虹图表；
- 数字弹跳（bounce ease）；背景动图；
- 无出处的数字上屏；时效戳缺失。
