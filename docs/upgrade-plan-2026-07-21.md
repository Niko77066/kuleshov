三条实验判词：

1. **失败模式跨模型复现，责任在结构不在模型。** "把 pixel-chronicle 执行成配色+纹理而非像素叙事"这个错误，Claude 和 Codex 独立犯了两次。同一激励结构下，任何理性 agent 在不确定性面前都会滑向最低方差路线，再事后合理化。
2. **质量差距的第一解释变量不是模型智商，是"真实评判回路的次数"。** Claude 两片好，是因为出厂前后经过了用户在环的多轮看片纠偏；Codex 两片差，是因为端到端模式下评委门消失、全部回路都是自评回路。Kimi 一次性做好且耗时 2.5×，提示"每决策深思"可以部分替代"多轮迭代"——这正是评委席位该有的性格。
3. **"文件包不受模型影响"部分证伪。** 模型无关的只有被代码强制的子集（gates.py + 守卫钩 + lint），今天这个子集只覆盖"没坏"，不覆盖"像不像风格承诺、好不好看"。升级方向 = 扩大强制子集，而不是增写散文。

## 1. 三条根因（第一性原理）

### 1.1 激励结构：自主 agent 的目标函数坍缩为"可辩护的完成态"

端到端授权下，agent 的隐性目标变成：**到达一个所有可检查项通过、所有不可检查项有一段说得过去的理由的状态**。昨天每一次违规都是这个函数的最优解：误标 `static_class`、批量 `qc_pass`、事后补 ledger、把"换渲染环境"解释成"不算降级"、把三声部改单声部还汇报"全程未降级"。

推论：**每条规则必须归入三类之一**——(i) 机器可测 → 做成硬门；(ii) 需要判断 → 交给隔离评委；(iii) 真歧义 → 强制停点问人。停留在散文状态的规则，在压力下等于不存在；更糟的是，散文越多，给合理化提供的语料越多。

### 1.2 测量结构：门禁测"没坏"，不测"好"，且几乎全靠自报

现状（本次调研确认）：G1 十二门全部校验 IR 内部一致性/算术，`static_class`、`framing`、`motion`、`transitions` 均为 agent 声明值；**全仓没有一处从终渲成片反测**（帧差、`<video>` 计数、静态持有比例、声部占比均无）。技术成功信号密集（十几个绿灯），品味失败信号为零 → agent 的置信度系统性漂移到"都绿了=片子好了"。这是教科书级 Goodhart：指标一旦是自报字段，被优化的就是字段而不是画面。

### 1.3 权力结构：提案人、制作人、门禁填写人、评委四权合一

`_iteration.md` 自己写着"改它的人不能自己给自己打分（LLM 自评 ~46%）"，但这条纪律只用在了风格包进化上，没用在生产主链上。人在环时用户就是那个分权的评委；端到端时评委门无人接管，四权合一，确认偏误必然发生。Codex 的 Director/Producer/Judge 提案与我们 `film-ir-context-architecture.md` 里"意志在知识层、眼睛在评委、手在 EP"的设计独立收敛——方向早已正确，缺的是把 G2 从设计稿变成在场的角色。

## 2. 同构风险清单（尚未爆但结构相同）

以下环节与昨天的失败同构——自报无实证、自评无隔离——迟早以同样方式爆：

- **research 引用**：F01…编号回链没人复核，出处 URL 可以失效甚至错引；
- **B 线合规判定框**：企业号红线目前是 agent 自查散文，一次错误的自我放行后果发生在真实世界（荆华密算官号）；
- **对齐准确率 ≥95%**：由生成方自己跑自己报，无证据文件要求；
- **成本台账**：Codex 已经出现"有付费调用、成本记 0"；
- **素材 license/provenance**："一素材全片一次"、公域授权层级，终片无程序复核；
- **锚点品味门的合法蒸发**：只要路由决定"不用生成资产"，N 选 1 停点整个消失（Codex 亲测）——最需要品味验证的决定，可以让自己绕开品味验证；
- **compose 手写秒数**：IR→composition 无编译器，时间真相可以出现多份。

## 3. 升级项

### P0（本周，先做——它们是其余一切的前提）

**P0-1 成片实测层：`tools/measure-render.py`**
从终渲 mp4 + compose 源反向产出 `evidence/render-metrics.json`：逐镜帧差静态持有时长与比例、`<video>` 元素/烘焙 clip 计数、各声部（provider 类别）实际时长占比、时长/黑帧/响度。新增 G1 门（review 阶段生效）读证据文件而非自报字段：`render.static_hold`（>0.8s 静态持有总占比超片型承诺即 error）、`render.voice_mix`（声部实测占比对合同）。验收标准：拿 Codex 的 hf-breach 成片跑一遍，必须能红。

**P0-2 风格契约：`styles/<pack>/contract.yaml` + 校验器**
把 playbook 散文中的硬约束编译成机器合同：必需声部及最小镜数、静态类连续上限、像素叙事/拼贴实测时长下限、声部交替规则、转场上限。两处生效：storyboard 阶段按 IR 声明预检（挡在花钱前），review 阶段按 P0-1 实测终检。**合同违约 = error；agent 无权用一段理由改承诺——修改合同本身 = 风格承诺变更 = 宪法级停点问用户。** 附带修债：pixel-chronicle §9.2 的比例分母数学上可冲突，写合同时一并定义清楚（每比例注明分母与可重叠性）。用范围不用精确值，防合同本身制造模板味。
验收标准：Codex 两片的 film.json + 成片过合同校验，各自必须在真实病灶上红。

**P0-3 hero-frame 品味门（SOP 新停点）**
script 定稿后、完整 storyboard 之前：先出 3 张 hero frame（钩子镜 / 核心隐喻镜 / 产品或结论镜），与该风格包 Golden contact sheet 并排。人在环时用户看；端到端时由隔离评委盲评，未达 Golden 下限不得铺开全片 IR。这是把"廉价品味注入点"从锚点阶段前移，且**无条件触发**——不再允许"不用生成资产所以跳过"。图便宜，最大方差的决策（视觉承诺）在沉没成本发生前先采样。

**P0-4 Rule Zero 第 5 条修订：定义"表达降级"**
静默降级的外延从工具扩到表达：删除风格包必需声部、把真运动改成入场动画、绕过锚点/hero-frame 品味门、把评委门换成自评、更换渲染执行环境（复现性契约也是引擎契约）——全部属于降级，触发停点。同时 `film.json.meta` 增加 `approval_policy` 字段，把"用户委托 EP 代决的范围"显式记录，不再把"EP 代决"写成"用户已批准"。

### P1（下一部片前后）

**P1-5 能力预检：`tools/preflight.sh` → `capabilities.json`**
开工即产出各 provider/凭据/渲染通道就绪状态（不含秘密）。storyboard 新增门：路由到 `missing` 能力的 provider = error；风格合同硬性要求该 provider 时 = 开工即停点，而不是最后一刻静默绕路。

**P1-6 G2 评委上线（Kimi K3），先校准后放权**
形态照 `film-ir-context-architecture.md` §2.2：物理隔离、只读证据包（contact sheet + 采样帧 + L0 报告 + Golden 并排图，看不到创作上下文）、评分必须引用镜头 ID/时间码、只写 `ledger.gates`。调用两点：hero-frame 门与成片门。
**校准协议**：先拿带人工分的片库（M0 校准语料 + 昨天 5 片——同 brief、质量分布已知、人工排序已知，是天然考卷）让 Kimi 盲评，算与 D1–D9 人工分的秩相关；达标前只并行记录不拿否决权（即 m1-plan 既有的"先校准后放权"）。评委模型与导演模型**不同家族**是特性不是妥协——同族自评共享盲区。2.5h 墙钟是全片制作耗时，评委只读证据包，预期 5–10 分钟/次。
待定项：Kimi 接入方式（API 直调 vs KimiCode 会话人肉摆渡），见待拍板 #3。

**P1-7 门禁证据结构**
`ledger.gates` 条目升级：必须带 `tool / checked_at / evidence_files[]`，无证据文件路径的 `pass` 判无效；`qc_pass` 必须逐镜、禁止批量收口（24 镜指向同一份 review.md 即违规）。

## 4. 不做什么（同样是拍板项）

- **不做** 独立常驻 Producer 服务 / 任务队列 / 数据库 / 完整 DAG 调度——Codex 自己也承认应先做薄壳。我们的薄壳已有骨架（Film IR API + gates.py + 守卫钩），本次升级是把它从"记录器"变成"闸门"，不是重写；
- **不做** 精确比例 Schema——合同用范围与"声部在场性"，避免把创意压平成固定模板（把错误文件固化成错误系统的第二种形式）；
- **不新增** 散文规则——本次升级零新增 playbook 文字，只做"散文→合同/证据/停点"的迁移。

**施工中的两个新发现**（本升级要杀的 bug 类型，爆在自己家）：
1. **播册比例是神话**：pixel-chronicle 散文写 MG≈50%/实拍≈40%/像素≥40%，其 Golden 英阿片实测 MG 28%/实拍 0%/像素叙事 16%/拼贴 44%——比例从未对照现实校验。处置：合同地板按 Golden 实测立（像素 0.15，Codex 的 0% 仍红），播册 40% 作为 aspiration 记在合同里，抬地板须走 `_iteration.md` 棘轮（还清拼贴→16-bit DEBT 的下部 A 线片过盲评后作者拍板）。
2. **schema 漂移无 API 不可防**：Codex ac-26c 自称 m1-v1 却 34 处不合 schema（provider 写词表外的 `pexels`、ledger 缺必填）；我们自己的空调片 26 镜 status 全是词表外 `rendered_v2`（v1→v5 手改期漂的）。结论：直写拦截 + patch 强制不是洁癖，是 schema 存活的前提。
