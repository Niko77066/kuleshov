## 二、最主要的原因：**沉淀是"文档"，不是"机器"**

这次真正拦住问题的门——film.json 守卫钩、`kuleshov-ir validate`、`hyperframes check`、旁白 linter——**没有一个失手**。所有踩的坑都发生在"规则只写成散文、靠人临场想起来去读"的地方。

- **知识是"按需加载"的散文，路由靠人。** 我加载了每阶段"显然该读"的参考，漏了**跨阶段/条件触发**的（server-render 只在"切默认"时、bgm 只在"要 BGM"时、woff2 是埋在参考里的新规）。
- **沉淀越多散文，越容易漏**——不是加载少了，是"该在此刻开火的规则"没有绑定到"此刻"。
- **一句话：写下来 ≠ 被执行。散文不会自己开火，只有代码会。**

次因：
- **清单只编码"旧疤"**（视觉自查 15 项是历次结痂），docsrc/s03b/av-sync 这些**新失败模式**不在表上，自查是"对着清单打勾"而非"挑剔观众会恨哪"。
- **"端到端"把便宜的品味门碾平了**：collage Gate1（隐喻停点）本是给用户 1 秒拍板的地方，正能拦下 s03b。端到端 ≠ 不暴露；贵/可逆的不停，但便宜的创意赌注（隐喻/BGM 基调/封面）仍要一行话异步抛出。

## 三、隐性知识（这次才显形，得写死）

1. **woff2 字体纪律**：compose 禁 `local("系统字体")` 承担正文/标题——否则渲染机 font-kit 兜底→长标题换行漂移（首片双跑 SSIM 0.981 的病根）。用 `SansSC/SerifSC` woff2；`font-weight` 只能取 woff2 实际有的档（900→700），否则本地/机器 faux-bold 合成不一致。
2. **渲染机比本地 docker 快 ~2.3×**（177–189s vs 419s），并发 1、503 要退避重试。双跑验收：全片 SSIM 会被拼贴视频 h264 重编码拉低（卡片段 0.986 > 拼贴段 0.970），**判据看关键版式帧目检（换行/安全区）逐像素一致，不看全片 SSIM 数字**。
3. **BGM 床要纹理不要旋律**：软 pad+低 drone、无主旋律/无鼓/无节拍、平淡；压旁白下 ~18–22 LU；prompt ≤300 字、`is_instrumental:true`。有性格的元素（tick/glitch/melody）会抢人声。见 `produce/references/bgm.md`。
4. **元素与容器边框的重合**是版式硬伤，`hyperframes check` 的包围盒重叠**查不出**（边框是 CSS border 不是元素）——必须抽帧目检 or 专门断言。
5. **拼贴隐喻必须复述口播的"主张"**，不能只贴"关键词"：口播是"从 AI 输出反推隐私"，画成"碎片拼人脸"就漂成了隔壁（重识别）。
6. **发布包不再随片出厂**（用户取消，运营端自理）。
7. 执行铁律（早写在参考里，仍翻）：APIhub/Seedance/fal 轮询**一律 curl**（网关封 urllib）；film.json **只走 `kuleshov-ir patch`**（守卫钩）；heredoc 里 ffmpeg 加 `-nostdin`；python heredoc 里 shell 变量要在 python 内重新定义。

## 五、对"更端到端"的判断

端到端是目标，但**自主度是乘数**：底下是硬门→放大质量；底下是散文→放大漏洞。这次最大化端到端跑，恰恰**放大了**所有未固化的缺口，逼用户在昂贵的复盘轮里逐个揪。结论：**先建执行层，再加自主度**。这次每个疤都是执行层的规格说明书——转成门，M0→M1 就在正常运转；不转，下片用同样的方式更快更贵地再翻一遍。
