## 反馈 2 · Film IR / 上下文范围 / IR API / 风格包定义 —— 已成文

全部在 **[docs/film-ir-context-architecture.md](film-ir-context-architecture.md)**。给评审的一段话摘要：

- **Film IR 文件系统**：一片一目录，`film.json` 是唯一真相源（编译器类比：阶段是前端、门禁是优化器、compose 是后端——像素贵且不可编辑，IR 便宜且可手术）。每个文件恰好一个 owner 阶段，过门即锁定。
- **导演—子导演上下文**：导演是三重存在（意志在风格包与宪法、眼睛在评委、手在 EP）。十阶段里有三处**全局屏障**（script / audio_timeline / storyboard 定稿）必须单 agent 持全局上下文，其余按逐镜头 DAG 并行。M1 只把三类席位 subagent 化：评委（要隔离）、motion（要广度）、anchors（可并行）——每席位一张上下文配给表（注入什么/按需读什么/写哪里/回传什么），纪律是"传 ID 不传本体、任务卡四要素、锁定后拿干净上下文进场、评委与创作者物理隔离"。
- **IR API 必要性**：M0 单写者不需要（约定够用）；M1 四个触发条件同时到达——motion 并行多写者（lost update 是并行的定义，不是假设）、Rule Zero 从自觉变强制、承诺字段禁静默修改、3 分钟片 film.json 超 2000 行的上下文经济。形态是 4 个正交动词的 CLI（`ir read/patch/validate/execute`），**不是** MCP server、不是 DAG 引擎（pull-based：没到触发条件不建）。工程化注意点 9 条见正文，头三条：写入口唯一化靠 hook 保证（提示词是 advisory，hook 才是 guarantee）、校验在写入层且与 G1 硬门共用代码、并发用"分区 + 乐观锁 + append-only ledger"（单机文件系统，禁止造飞机）。
- **风格包定义**：不是 skill，是**版本化的品味数据资产（SKU）**——skill 是行为（跨风格共享的流程），风格包是参数（这次穿哪套手艺）。评审问的"EP 确认风格包后 spawn subagent 时写入上下文"**方向正确，三处修正**：① 按席位裁剪切片注入，不整包平铺（motion 只拿 Seedance 方言行，compose 只拿视觉系统）；② 能编译的不注入（CSS tokens 直接进 composition 源码、LUT 进渲染参数、可判定黑名单项进校验器）；③ 切片逐字注入不许转述 + `style_pack@sha` 记版本。评委拿同一份 rubric 附录——创作侧没贯彻，评委按同一标准拦，风格包从"建议"升级为闭环约束。

## 反馈 3 · 审美/创意评估 + 渲染前评估 —— 立场 + 三个补强

**立场（蓝图 §9 已有，向评审讲清楚即可）**：开放式审美判断不可校准——"agent 看片打分"是全体系最不可靠的一环，所以审美主要**靠构造注入，不靠检查把关**：品味宪法与风格包是审美的供给侧（考片制准入：样片盲评赢基线才进库）；评委只回答封闭问题"这是不是合格的我们家的片"（风格依从可锚定、可校准），不回答"这是不是好电影"。创意的评估点在花钱之前：blueprint 评委门评概念差异化，出厂前跑"换个选题/产品名还成立吗"模板味审查（品味宪法条目）。

**承认反馈有效，三个补强（M1 落地）**：

1. **评分卡补维度**：现行 7 维（意图/准确/开头/节奏/音画/字幕版式/完成度）确无显式审美与创意维——增补 **D8 风格依从**（对照风格包视觉系统+黑名单逐条核对，锚点描述可写死）与 **D9 原创性/模板味**（蓝图 L4 评委 rubric 本就含原创性维）。先积累人工分，评委后校准。
2. **L1 美学预测器落地**：对锚点图与成片抽帧跑美学预测分（蓝图 L1 层已列）——用于**排序与预警**（多候选选优、掉分告警），不做一票否决（不可校准的仪器不给否决权）。
3. **审美靠"选出来"**：关键镜头（hook/hero/角色）并行 2 候选 + 评委/用户选优——成本 +1 次生成、墙钟归零，是全自动系统里最廉价可靠的品味注入点（anchors 阶段的 N 选 1 已在 SOP 里，M1 延伸到高风险镜头）。

**"片段产出后先评估、避免渲染浪费"——已是设计核心，直接回答"有"**：评估金字塔的门禁经济学就是这个——花钱动作从便宜到贵排成阶梯（文本 → 图 → 单镜头视频 → 全片渲染），每级过了上级全部门禁才解锁。具体到执行顺序：storyboard 八项自查拦在**花任何钱之前**；锚点先行拦在视频生成之前（图便宜，多轮迭代在图上做）；**镜头级 QC 在每段烘焙 clip 落地时就跑**（技术层 ffprobe/黑帧 + 语义层对照 shot intent），不合格重试 ≤2 次、不进 compose——坏镜头不会被合成掩埋；compose 的 lint 不过禁 render。M1 补强：3 分钟片最终渲染墙钟长，加**代理渲染预览**（低清 preview + VLM 走查）挡在 `--docker` 帧级复现渲染之前；配合 `ir execute` 的受影响集计算，重做永远是"单镜头/单区间"，不是整片。

## 反馈 4 · 里程碑（明天 M1、下周三 M2）—— 诚实评估：有难点，需要先对齐"完成"的定义

**首先要对齐验收口径**：蓝图里的 M1 是"~20 条金样本全自动出厂 + 资讯速览连续 7 天无人值守日更"，评审语境的 M1 是"两个目标片种能通过 pro 链路生产出来"。**前者明天不可能，后者明天也做不完但能开起来。**建议向团队明确：7/17 交付的是"M1 首片开拍 + 工程化骨架"，M1 的完成判据改为"两个目标各出厂 ≥1 条、过评分卡、带发布包"。

**四个真实难点（按风险排序）**：

1. **实拍素材通道缺口（最硬）**：对标片 40% 是纪录片级实拍素材，而检索剪辑 provider 排在 M3（语料库、授权体系都未建）。M1 三个替代路径，开拍前必须选定（elicitation 已写进 pixel-chronicle 风格包第 3 问）：人工投喂授权素材（最稳，靠人）；Seedance 生成纪录风空镜 + 强风格化缝合（对标片自己就用 AI 图强风格化规避塑料感，路由上合法）；或首片选实拍依赖低的选题。**不选定就开拍，会在 motion 阶段撞墙。**
2. **体量跳档**：M0 实测最长 90s/16 镜（《800V》），对标片 4 分钟 ≈ 1200 字文案、40 镜当量——脚本、分镜、生成、QC 全链条工作量 ×2.7，人在环确认轮次同步变多。首片墙钟按"一个工作日跑不完"预期，排到周末出厂是诚实排期。
3. **工程化与内容生产抢同一天**：IR API、subagent 任务卡、G2 评委、发布包工序不可能与两条片同日完成。建议明天只做两件工程活：`schema/film.schema.json` + `ir read/validate`（半天，立刻服务首片），其余按 [落地顺序](film-ir-context-architecture.md#5-落地顺序m1-内) 排进下周。
4. **荆华密算依赖外部输入**：品牌素材、产品事实底座（功能边界、合规资质表述口径）需要业务侧给料；内容全部过 compliance.md 判定框。建议荆华密算首片排 M2 窗口（下周），M1 窗口先跑通对标片。
