# 选题系统

> 按需加载：场景 B。产出规格：每次 5 个候选，各带目标搜索词、预期封面文案、系列归属。

## 方法一 · 搜索词反推（小红书的根本打法）

小红书 60%± 流量来自搜索、笔记长尾数月 [社区共识]——选题第一问不是"我想讲什么"，是"用户在搜什么"。

1. 在小红书搜索框输入领域主词（"隐私"“病历”“聊天记录”），收集下拉联想词与"大家都在搜"；
2. 优先做**长尾问句词**："AI 会偷看我的聊天记录吗"“简历投出去会被卖吗”“心理咨询记录保密吗”——问句词 = 用户原话 = 精准意图；
3. 七三原则：70% 选题做长尾词（稳定长尾流量），30% 追热点事件（某泄露事件上新闻当天出解读）[社区共识]；
4. 每个选题的目标搜索词记入 `user-data/<账号>/strategy.md`，发布后验证收录与搜索流量占比——形成"选题 → 验证"闭环。

## 方法二 · 科普选题四象限（移植自 x-mentor 的 4A，按科普语境改造）

| 象限 | 问题形态 | 例 |
|---|---|---|
| 可行动 Actionable | "怎么办" | 三步关掉 App 的麦克风权限 |
| 可分析 Analytical | "为什么/怎么回事" | 你的病历是怎么流到保险公司的 |
| 反常识 Counter-intuitive | "居然是这样" | 删掉的记录其实没删 |
| 有共鸣 Anthropological | "我也遇到过" | 被大数据杀熟的一天 |

健康配比 [假设]：可行动 40% / 可分析 30% / 反常识 20% / 共鸣 10%。连续 3 期同象限 = 选题在偷懒。

## 方法三 · 荆华密算内容引擎（企业号专用，2026-07-16 定稿）

**主引擎是热点驱动**：搜集全球 AI 隐私与泄露热点新闻（数据泄露事件、AI 训练数据争议、监管动态、平台隐私政策变更）→ 专业解读 + 安全提醒 → 落到六场景中的产品价值。工序：

1. 热点池：每日扫全球隐私/泄露新闻源，按"话题热度 × 与六场景的贴合度 × 可解读深度"三票筛选；
2. 解读框架（每条固定三段）：**发生了什么**（事实，带出处）→ **对你意味着什么**（场景化风险 + 安全提醒，过 compliance.md 判定框）→ **制度/技术上怎么防**（产品价值只在此段落点，占比克制）；
3. 时效纪律：热点解读 48h 内出片（同 news_recap 时效硬门）；错过窗口的热点降级进常青库。

**下面的场景矩阵是常青补充库**（热点空窗期取材 + 合集连载的骨架），六场景 × 三框 = 18 个格子，排期从格子里取，保证不重复不跑偏：

| 场景 | 风险故事框（个人号优先） | 法规解读框（企业号主场） | 产品价值框（≤1/5 占比） |
|---|---|---|---|
| 法律 | 律师函里的隐私条款翻车 | 《个保法》逐条讲 | 密算如何做合规设计 |
| 医疗 | 病历流出的一次事故复盘 | 病历数据的法定边界 | 医疗数据的密态计算 |
| 心理 | 咨询记录被第三方看到 | 心理咨询保密义务 | 敏感对话的加密方案 |
| 职场 | 背调查出了什么 | 员工监控的合法边界 | 企业侧合规工具 |
| 金融 | 征信查询记录泄露 | 金融数据规范 | 密态风控 |
| 科研 | 数据集里的真人信息 | 科研数据脱敏义务 | 隐私计算于科研协作 |

纪律：产品价值框占比 ≤ 1/5（企业号发硬广是黑名单项）；风险故事框全部过 `compliance.md` 判定框（讲风险不给个案建议）；每个场景做成合集连载。

## 系列化优先

两平台都奖励系列（小红书合集连续放送、抖音 7 天推流期 + 系列化价值上升 [社区共识]）。新选题先问"挂进哪个系列"：科普号如《隐私翻车现场》《个保法逐条讲》；单篇孤立选题要有单独立项理由。
