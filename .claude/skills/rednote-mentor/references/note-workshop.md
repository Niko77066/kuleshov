## 封面公式（小红书 3:4 文字封面）

小红书的封面 = X 的 Hook = 抖音的前 3 秒。生死线，独立打磨，禁止用视频截图凑数。

1. **大字报式**：一句痛点/结论顶满画面（≤8 字），底图弱化——知识类 CTR 最稳 [社区共识]
2. **对比式**：泄露前 vs 泄露后 / 你以为 vs 实际上，左右分屏 [社区共识]
3. **清单式**："3 个习惯正在出卖你的病历"——数字 + 名词 [社区共识]
4. 全账号统一模板（配色/字体/版式），让用户在瀑布流里认出你 [社区共识]——对接 Kuleshov 风格包的视觉系统（CSS tokens 直接产封面）
5. 真人出镜封面 CTR 普遍更高 [社区共识]——科普号可用固定人设图，企业号用场景图

## 标题公式（≤20 字，含 1 个目标搜索词）

结构 = 人群/场景词 + 痛点词 + 承诺（数字或反常识）：

- 痛点提问："看病记录会被保险公司看到吗？"（场景词"看病记录"即搜索词）
- 数字清单："医院泄露你隐私的 3 个环节"
- 反常识："删掉的聊天记录，其实还在"
- 法规锚（荆华密算专用）："《个保法》第 45 条：你有权拿回你的病历"

禁：标题党无兑现（承诺什么正文必须给什么——评论区"就这？"是负面信号）；无任何可搜词的抒情标题。

## 视频脚本结构（3 分钟知识科普）

```
0–3s    钩子：具体事故/反常识断言/直给痛点（抖音生死线）
3–15s   核心承诺："这条视频讲清楚 X 的三个环节"（抖音 15s 铁律）
15s–2m40 干货主体：每 15–30s 一个小高潮（新论点/新画面/新数据）
          ——与品味宪法"每 8–10s 一次视觉变化"兼容：视觉变化更密，信息高潮稍疏
最后 15s  收藏理由（"存下来，办住院手续前看一遍"）+ 追问式提问引评论 + 合集追更引导
```

小红书版与抖音版差异最小改造：前 3 秒重做（抖音直给冲击，小红书可稍缓）、封面独立、标题各按平台公式。

## 发布包工序（视频出厂时随片交付）

存 `projects/<片名>/out/publish/`，双平台各一份：

| 件 | 小红书 | 抖音 |
|---|---|---|
| 封面 | 3:4（1080×1440）文字封面，≤8 字 | 竖版首帧图（9:16） |
| 标题 | ≤20 字，含目标搜索词 | 含搜索词，可稍长 |
| 正文/描述 | 400–600 字，埋长尾词，结尾引导收藏 | 简短 + 埋词字幕依赖视频本体 |
| 话题标签 | 3–5 个，为搜索收录服务 | 2–3 个，为兴趣归类服务 |
| 发布时间 | 目标人群活跃时段（通勤/午休/睡前），账号实测为准 [假设] | 同左 |
| 目标搜索词 | 记录进 strategy.md，发布后验证收录 | 同左 |

## 质量清单（场景 A Step 4 / 场景 C 自动跑）

1. 目标搜索词：标题含 1 个？正文自然出现 2–3 次？
2. 封面独立设计？文字 ≤8 字？与账号模板一致？
3. 前 3 秒有钩子、15 秒内有核心承诺？（视频类）
4. 结尾有收藏理由 + 评论引导？
5. 违禁词扫描：绝对化用语 / 虚假承诺 / 引流词 / 医疗断言（企业号加跑 compliance.md 判定框）
6. 承诺兑现：标题许诺的，正文给全了吗？
7. 系列归属：能挂进哪个合集？没有合集可挂的选题要说明为什么值得做单篇
8. 双发检查：两平台版本至少封面/标题/前 3 秒不同
