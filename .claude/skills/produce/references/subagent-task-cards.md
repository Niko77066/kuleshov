# 子导演任务卡模板（导演分派 subagent 用）

> 依据 `docs/film-ir-context-architecture.md` §2.2 席位配给表。生产 session 里 EP spawn subagent 时按模板填空。
> 四条纪律：传 ID 不传本体；任务卡四要素（目标/输出/工具/边界）；锁定后拿干净上下文进场；评委与创作者物理隔离。
> **M0 写权限收敛**：子导演不直接写 film.json——回传固定格式摘要，EP 主循环落盘并记 ledger。IR API 上线后改为 `ir patch --actor <席位>` 直写。

## 通用规则（三张卡共用）

- **注入 = 逐字复制**，EP 不许转述/概括风格包与 IR 切片（概括必走样）；每张卡首行带 actor 署名。
- 回传 ≤2000 token，格式固定（见各卡）；EP 不读子导演完整过程。
- 任何被迫的参数改变（超时想丢首帧、引擎不可用想换路）：**停下回报，不许自决**（铁律 5）。

## 卡 1 · motion 子导演（每镜头组一张，可并行 fan-out）

```
actor: motion:<group_id>
目标：生成镜头组 <group_id>（镜头 <s05,s06,s07>）的烘焙 clip，并逐镜自查 QC。

注入（按序贴入）：
1. 本组镜头规格 —— film.json 的 shot_groups[<id>] + 所属 shots[] 条目原文（含 t/intent/framing/source/anchor_refs/接缝契约）
2. 锚点清单 —— anchors/ 下相关文件路径 + anchors[] 条目原文
3. 引擎知识包 —— references/seedance.md（数字人组换 avatar.md）全文
4. 风格包切片 —— 该包 playbook 的「节奏与剪辑」「声部表」相关行 + 翻译总表引用行原文

边界：不许改剧本/分镜/路由/时间区间；不许触碰本组之外的任何镜头；旁白轨永远不由生成模型产出。
资源档位：单镜重试 ≤2（锁 seed 改 prompt）；到限即停、如实回报。
产物落盘：shots/<group_id>*.mp4（+ 尾帧提取，若接缝契约为 A 型）。

回传格式：
- 状态：done / partial / blocked
- 逐镜：实际时长 / 技术 QC（黑帧/分辨率）/ 语义自查（intent 兑现、畸变、锚点一致）/ seed / 完整 prompt / 成本 / 墙钟
- 异常与 DEBT：<无则写无>
```

## 卡 2 · anchors 子导演（可并行）

```
actor: anchors:<batch_id>
目标：产出锚点批次 <角色表/风格帧/关键帧清单>，参数全记录。

注入：
1. 锚点需求清单 —— film.json anchors[] 待产条目原文（类型/用途/构图要求）
2. 引擎知识包 —— references/image-motion.md 全文
3. 风格包切片 —— 该包 playbook 「视觉系统」全节（CSS tokens/色板/版式）+ 反模式黑名单

边界：不许改锚点需求本身；风格帧色板以风格包 tokens 为准，不许自创配色。
资源档位：每锚点 ≤3 轮迭代（图便宜，但也有限）；每锚点产 2–4 候选供 N 选 1。
产物落盘：anchors/<id>_v<n>.png。

回传格式：候选清单（文件路径 × 生成参数 × 一句自评）/ 一致性备注 / 成本。
```

## 卡 3 · G2 评委（物理隔离，禁止复用创作上下文）

```
actor: judge:<stage>:<n>
目标：按 rubric 对 <blueprint/storyboard/成片> 打分并给修改意见。

注入：
1. rubric —— 评分维度 + 1–5 分锚定描述（评分卡定义 9 维中适用维）
2. 风格包附录 —— 该包「风格宣言」+「反模式黑名单」（逐条核对用）
3. 锚片 —— 2–3 条已知人工分数的对照（golden set 摘引 + 分数）

输入（只给产物，不给过程）：artifact 路径 / 采样帧 / 转写文本 / L0 仪器报告。
边界：看不到创作过程与中间辩解；只回答"是不是合格的我们家的片"，不回答"是不是好电影"。
判定纪律：每维 1–5 分带理由；**分数必须引用具体位置（镜头 ID/时间码/行号），引用缺失判无效**；黑名单逐条核对，命中即一票否决项单列。

回传格式：逐维分 + 引用 / 黑名单核对结果 / 修改建议（≤3 条，按影响排序）/ 置信度（低置信标注，交人工）。
```
