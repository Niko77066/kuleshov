# 引擎知识包 · 视觉出厂自查（版式/画面反模式 linter）

> 何时读我：storyboard 自查、compose 自查、**⑨ review 出厂前必过**。
> 定位：脚本层有 `narration-voice` linter、拼贴层有 `collage-broll` 死尾 QA、技术层有 `hyperframes check`——本文件补上**视觉/版式层**的可执行自查，让出厂前 **agent 自己发现问题、自己指出**，不靠用户逐条挑。
> 由来：`openai-78m-logs` 的 daily-brief v1「全军覆没」尸检（2026-07-17，用户反馈）+ 品味宪法 + 竖屏官号经验。**每条都是踩过的坑。**

## 怎么用（可执行，不是建议）

抽帧过清单：`npx hyperframes snapshot --at <各镜中点>` 出 contact sheet（或成片 review 帧），**逐镜逐项**过下面硬查项。**任一硬查项命中 = fail，必改再出厂**，结论 + 命中项 + 证据帧写 `ledger.gates`。这是 ⑨ review 的一道门。

## 硬查项（一票否决 · daily-brief 尸检清单）

1. **竖屏视觉重心**：内容**居中、填满画面中部**了吗？有没有「上堆下空」（内容挤上方、下半屏大片留白）？→ 竖屏用户视线在中央，上堆下空 = 废镜。
2. **信息层级**：单屏信息元素 ≤ 2 个？有没有「左上 kicker + 左下 src 小字」那套**双角标**（PPT 味的直接来源）？出处并进画面或压到极轻，不抢视线。
3. **文件实证**：讲到**具体文件 / 法规 / 数据**时，画面有没有「读文件」（真截图 / 高保真复现 + 浏览器 chrome + 同步高亮）或数字 callout 压在文件图上？→ **提文件却用纯版式摆字 = fail**（参 samsung `s03-consent-read.html`）。
4. **禁死尾 / 冻结**：烘焙 clip（拼贴 / Seedance）尾段有没有 > 0.8s 近静止（`select='gte(t,dur-1.5)',signalstats` 的 YDIF < 1）？→ 冻结帧补时长是宪法零容忍。版式静态卡属 `typography_led` 本征，不算。
5. **幻灯片化**：连续同版式 / 同景别 ≤ 2 镜？静态类（纯版式 / 图片动效）连续 ≤ 2？每 8–10s 至少一次视觉变化？
6. **模板味**（对齐 D8）：换个选题 / 产品名 / 品牌名，这版画面还成立吗？**成立 = 没做够**。
7. **三面开钩一致**：封面文案 / 首 3 秒画面 / 口播首句 = 同一个承诺？
8. **质感缝合**：混排（拼贴 + 版式 + 文件）靠统一颗粒 / keyline / 点色缝成一家了吗，还是割裂成几段？
9. **文字不拆词**：大字标题有没有把词拆到两行（如「消/失」「匿/名」「办/法」）？→ nowrap 或缩字号。

## 硬查项 · 第二批（英阿片 v3–v6 五轮用户返工沉淀，2026-07-19）

10. **素材重复（零容忍）**：同一实拍素材全片出现 >1 次即 fail——构建期断言 `Counter(broll srcs) 无 >1`，出厂抽帧再人工复核"既视感"。宁可单段拉到 5s，不许复用画面。
11. **时代/地点错位**：叙事段的实拍素材与所述年代/地点冲突（1806 段出现汽车街景、"墨西哥城"配布宜诺斯艾利斯地标）= fail。逐叙事段问一句：**这画面放在这句旁白下，历史/地理上成立吗？**
12. **暗尾黑闪**：素材挂载窗触到源片末尾淡出段（观感=莫名黑屏）= fail。构建期 `media_start+dur ≤ 源长−0.8s`；轨道相邻 clip 断言无缝隙无重叠。
13. **音画同步抽查（协议化）**：全片均布 ≥6 个句首词，按 timeline 戳切 1.2s 音频独立转写——词不在切片开头 = 对齐层 fail（见 forced-alignment.md）。字幕出现/消失与口播听感差 >0.2s 即打回。
14. **时效词出厂复核（新增门）**：终渲染前 grep 旁白全文的相对时间词（昨天/今天/上周/上礼拜/刚刚/本周/明天），逐个对照 research 里的**绝对日期**与**预计发布日**核算是否仍准确——制作周期一长，"昨天凌晨"就是硬伤（英阿片 v6 实改）。改词=整条音轨重生成+对齐全链重跑，别抱侥幸。
15. **烘焙镜头拍点**：组装/动画类镜头的完成拍点（payoff 帧）是否落在对应文案上（尾对齐规则，seedance.md）？空纸场/空景撞关键叙事句 = fail。

## 五维自评（各 ≥ 4 才出厂，记 ledger）

版式重心 / 层级克制 / 实证充分 / 动静得当(无死尾) / 缝合统一。低于线必改。

## 与评分卡（评委）的挂钩

本清单是 9 维评分卡的**画面侧展开**——打分时对应维度直接引用本清单锚点：
- 硬查 1、2、9 → **D6 字幕与版式**；3 → **D2 信息准确 / D5 音画协调**（实证）；4、5 → **D4 节奏**；6 → **D8 创意**；7 → **D3 开头**；8 → **D7 完成度**。
- 命中任一硬查项，对应维度**扣到 ≤ 2**，并在 `/video-triage` 记一笔归因到环节（style_pack / storyboard / compose）。
